# Governance Lifecycle

Status: canonical-draft

## Purpose

Defines the canonical lifecycle for governance, treasury, protocol, application, and continuity actions.

## Lifecycle

1. Draft
2. Sponsorship
3. Review
4. Governance Vote
5. Timelock
6. Execution
7. Receipt

### Draft
Minimum eligibility: V2 Steward Candidate.
Creates a non-binding proposal.

### Sponsorship
Minimum eligibility: V3 Module Steward.
Moves a proposal into formal review.

### Review
Performed by the appropriate council.
Review evaluates but does not approve.

### Governance Vote
Binding governance participation by eligible governance stewards.

### Timelock
Required before execution.
Suggested ranges:
- Minor: 24h
- Treasury: 72h
- Protocol: 7d
- Constitutional: 14–30d

### Execution
Separate from voting.
Must occur through approved execution pathways.

### Receipt
ARCnet records factual governance events only.

## Treasury Flow
Draft → Sponsor → Treasury Review → Vote → Timelock → Execute → Receipt

## Protocol Flow
Draft → Review → Testing → Vote → Timelock → Upgrade → Receipt

## Constitutional Rule
No proposal may bypass review, governance, timelock, and receipt generation.
