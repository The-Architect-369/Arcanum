# Tempus Windows

## Purpose

Tempus renders cycles, calendars, windows, and rhythms.

It must offer opportunity without pressure.

## Rules

- Windows may open and close visibly.
- Participation may be recorded when the user chooses to act.
- Non-participation is neutral.
- Missed windows must not create shame, penalty, loss of dignity, or worth implication.
- The app must not frame timing as personal superiority.
- The app must not use stress-inducing countdowns.
- The app must not define readiness or becoming.

## MANA and Tempus

MANA may be used for optional utility, infrastructure, anti-spam, or event creation if explicitly priced and confirmed.

MANA must not:

- accelerate time
- bypass readiness
- buy Vitae recognition
- punish silence
- create a perfect-play economy

## Chain interaction

If Tempus touches ARCnet, it may emit factual receipts only.

Receipts may state that an action occurred. They must not interpret what that action means.
